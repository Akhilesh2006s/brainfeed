### 1.3 Color Application Rules

**Backgrounds:**
- Primary: White (#FFFFFF) or Gray-50 for breathing room
- Cards/Sections: Gray-100 or Blue-50
- Hover states: Gray-100 (light) or Blue-50 (Blue-heavy sections)
- Dark sections: Deep Navy (#1A3B52) with white text for authority/premium feel

**Text:**
- Primary text (body): Gray-900 on white/light backgrounds (21:1 contrast ratio = WCAG AAA)
- Secondary text: Gray-700 for meta information (12:1 contrast ratio = WCAG AA)
- Links: Primary Blue (#0066CC) - never use default blue
- Accent highlights: Teal and Orange (used sparingly, max 15% of design)

**Interactive Elements:**
- Primary CTAs: Primary Blue background, white text (action-oriented)
- Secondary CTAs: Teal background or outlined style with Blue border
- Success states: Green with white icon/text
- Warning/Error: Red with white text

**Data Visualization:**
- Charts use gradient: Blue â†’ Teal â†’ Green (showing progression/improvement)
- Status badges: Green (success), Orange (in progress), Red (blocked), Blue (neutral)
- Heatmaps: Use 5-color gradient avoiding red-green confusion for color-blind users

## PART 2: TYPOGRAPHY SYSTEM

### 2.1 Font Pairing Strategy

**Headings (Display & Section Titles):** Figtree (Bold 700/800)
- Modern, geometric construction
- Optimized for readability at all sizes
- Includes 18 weights for flexibility
- Excellent multilingual support

**Body Text (Articles, Descriptions):** Inter (Regular 400, Medium 500)
- Highly legible at small sizes
- Perfect for dense content
- Natural line-height proportions
- Open source and web-optimized

**Code/Technical Content:** JetBrains Mono (Regular 400, Medium 500)
- Monospace for tutorials and code snippets
- Distinctive character shapes prevent confusion
- Great for developer-audience content

**Fallback Stack:**
```
Headings: Figtree, 'Segoe UI', system-ui, sans-serif
Body: Inter, 'Helvetica Neue', sans-serif
Code: 'JetBrains Mono', 'Courier New', monospace
```

### 2.2 Type Scale & Sizing

Use a **1.125 modular scale** (golden ratio approximation) for harmonious sizing:

| Level | Font | Size | Line Height | Letter Spacing | Usage |
|-------|------|------|-------------|----------------|-------|
| **Display** | Figtree Bold | 48px | 1.2 (58px) | -0.02em | Hero headlines |
| **H1** | Figtree Bold | 38px | 1.25 (48px) | -0.015em | Page titles, featured articles |
| **H2** | Figtree Bold | 32px | 1.3 (42px) | -0.01em | Section headers |
| **H3** | Figtree Bold | 24px | 1.35 (32px) | 0em | Subsection headers |
| **H4** | Figtree Medium | 20px | 1.4 (28px) | 0em | Card titles, widget headers |
| **H5** | Figtree Medium | 18px | 1.5 (27px) | 0em | Form labels, small headings |
| **Body Large** | Inter Regular | 16px | 1.6 (26px) | 0em | Primary article text, large descriptions |
| **Body Regular** | Inter Regular | 14px | 1.6 (22px) | 0em | Secondary text, meta info |
| **Body Small** | Inter Regular | 12px | 1.5 (18px) | 0.02em | Captions, timestamps, footnotes |
| **Caption** | Inter Medium | 11px | 1.4 (15px) | 0.03em | Tags, badges, very small text |

**Responsive Adjustments (Mobile):**
```
Display: 32px â†’ 48px (scales up from mobile to desktop)
H1: 28px â†’ 38px
H2: 24px â†’ 32px
H3: 20px â†’ 24px
Body: 14px â†’ 16px (stays consistent for readability)
```

### 2.3 Typography Best Practices

- **Maximum line length**: 65-75 characters (optimal for reading comprehension)
- **Heading weight**: Always 700+ (bold) to create hierarchy
- **Body weight**: 400 regular (lighter fonts reduce fatigue on long reads)
- **Emphasis**: Use Figtree Medium (500) for important phrases instead of all-caps
- **Never use**: ALL CAPS for body text (use uppercase for small badges/tags only)
- **Letter spacing**: Increase for headings (-0.02em to -0.015em creates premium feel), decrease for body (0em)

- Largest Contentful Paint (LCP): < 2.5 seconds
- Cumulative Layout Shift (CLS): < 0.1
- Time to Interactive (TTI): < 3 seconds

**Image Optimization:**
- Use WebP format for modern browsers (25-35% smaller than JPEG)
- Lazy load all images below the fold
- Max image sizes:
  - Hero image: 400KB (1920x1080)
  - Article thumbnails: 80KB (400x300)
  - Avatar/profile pictures: 30KB (200x200)
- Implement responsive images (srcset for different screen sizes)

**CSS & JavaScript:**
- Critical CSS inlined (< 50KB)
- JavaScript split into chunks (100KB max per chunk)
- Use CSS-in-JS for component styling (avoid external stylesheets)
- Cache busting through versioning

### 7.2 Accessibility Standards (WCAG 2.1 AA)

**Color Contrast:**
- Normal text: 4.5:1 ratio minimum (Blue-600 on white = 5.2:1 âœ“)
- Large text (18px+): 3:1 ratio minimum
- Test all text colors using contrast checker

**Keyboard Navigation:**
- Tab order follows visual flow (left to right, top to bottom)
- All interactive elements accessible via keyboard
- Focus indicators: 2px outline with 3px offset
- Skip navigation link (top of page, keyboard-accessible)

**Alternative Text:**
- All images: descriptive alt text (not "image123.jpg")
- Decorative images: alt="" (skip from screen readers)
- Example: âœ“ "Student working on AI chatbot project" âœ“
- Example: âœ— "student_image_01.jpg" âœ—

**Color Independence:**
- Don't rely solely on color to convey information
- Example: âœ“ "Success (green checkmark)" âœ“
- Example: âœ— "Green for success" âœ—
- Charts: Use patterns + colors (red + diagonal lines)

**Motion & Animation:**
- Respect `prefers-reduced-motion` media query
- Disable animations for users who set this preference
- No auto-playing videos with sound
- No flashing content (> 3 times per second)

**Mobile Navigation:**

**Form Inputs on Mobile:**

- Hero headline: 48px (largest)
- Article titles: 24px
- Body text: 16px

**Color & Contrast:**
- Primary content: High contrast (dark text on white)
- Secondary content: Medium contrast (Gray-700 on white)
- Tertiary content: Low contrast (Gray-500 on white)
- CTA buttons: High saturation color (Primary Blue)

**Whitespace:**
- Around key elements to draw attention
- Between sections: 40-64px
- Between cards: 24px
- Around text: Generous margins (16px min)

**Proximity & Grouping:**
- Related items grouped together with 8-12px spacing
- Different concepts separated by 24-32px
- Logical content sections: 40-64px gap

**Visual Weight:**
- Bold = important (headlines use Figtree Bold)
- Large = important (featured content is largest)
- Bright color = action-oriented (blue CTAs stand out)
- Full width = prominent (hero section, email CTA)
